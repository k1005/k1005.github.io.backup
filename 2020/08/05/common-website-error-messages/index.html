# 가장 일반적인 7가지 웹사이트 에러 메시지

https://www.siteuptime.com/blog/2018/06/15/seven-common-website-error-messages/ (opens new window)

웹사이트 소유자는 이미 웹사이트에 오류 메시지가 표시되는 것을 보았을 것입니다.

사이트에 오류가 있는지 모니터링하고 해결 방법을 아는 것이 중요합니다.

어떤 경우에는 사이트를 잘못 탐색하는 사용자의 잘못입니다. 그러나 일부 작업을 수행하여 제거할 수 있는 몇 가지 오류가 있습니다. 깨진 링크나 URL을 수정하는 것만큼 쉬울 수 있습니다.

이러한 각 오류에는 모든 내용을 자세히 설명하는 코드가 첨부되어 있습니다. 이것은 오류를 해결하고 사이트를 백업하고 작동시키는 열쇠입니다.

가장 일반적인 것이 무엇인지 알고 싶으십니까?

7가지 일반적인 웹사이트 오류 메시지와 그 원인을 알아보려고 합니다.

# 1. 500 Internal Server Error

500 Internal Server Error는 가장 일반적인 웹 사이트 오류 메시지 중 하나입니다.

일반적인 목적으로 사용되는 포괄적인 오류 메시지입니다. 기본적으로 다른 오류 코드가 적용되지 않거나 프로그래머가 문제를 식별하지 못한 경우에 발견됩니다.

웹 서버에서 내부 문제가 발생할 때마다 메시지가 트리거됩니다. 일반적으로 서버가 너무 열심히 작동하고 과부하가 발생했음을 의미합니다. 따라서 요청을 쉽게 처리할 수 없습니다.

서버가 다운되면 일반적으로 비즈니스와 사이트 모두에 나쁜 소식입니다. 시간과 비용이 모두 들 수 있습니다. 가능한 한 빨리 수정하십시오.

사용자가 시도를 다시 시도하고 다른 응답을 받을 수 있습니다. 그러나 특히 사이트에서 돈과 관련된 무언가를 실행하는 경우 큰 문제가 될 수 있습니다. 사용자가 장바구니로 요청을 다시 시도하면 중복 주문이 생성될 수 있습니다.

# 2. 404 Not Found

404 오류는 사용자가 접하는 일부 웹 사이트 오류 코드 목록에 따르면 가장 자주 발생하는 오류입니다. 누군가가 존재하지 않는 페이지에 접근하려고 할 때 발생합니다. 원인이 될 수 있는 몇 가지 다른 원인이 있습니다.

  1. 링크가 끊어졌습니다.
  2. 해당 페이지가 이동되었습니다.
  3. 페이지가 완전히 삭제되었습니다.

간단히 말해서 요청한 리소스를 찾을 수 없거나 사용할 수 없습니다.

이것은 일반적으로 쉬운 수정입니다. 대부분의 원인은 원인은 사용자가 페이지 URL에 오타가 있기 때문입니다. 잘못된 도메인 이름을 입력했을 수도 있습니다.

또 다른 이유는 오래된 페이지를 사용하는 경우입니다. 누군가가 귀하의 블로그에 기사를 작성한 경우 페이지에 링크되었을 수 있습니다. 해당 페이지가 더 이상 존재하지 않으면 404 오류가 발생합니다.

이러한 404 오류는 사용자 경험에 부정적인 영향을 미치므로 주의하는 것이 중요합니다.

정통한 사이트 소유자에게는 페이지를 기회로 사용할 수 있습니다. 표준 "페이지를 찾을 수 없음"을 사용하지 말고 404 페이지를 유용하게 만드십시오. 사용자가 도달하려고 했을 수 있는 페이지에 대한 제안 목록을 포함합니다.

# 3. 408 Request Timeout

408 오류 메시지는 종종 사용자가 페이지를 닫을 때 발생합니다. 즉, 서버가 정보를 검색하기 전에 사용자가 요청을 조기에 중단했음을 알리는 코드입니다.

링크를 너무 빨리 클릭하거나 버튼을 눌러 페이지 로딩을 중지하는 것도 원인입니다. 이 메시지는 매우 큰 파일과 관련하여 또는 서버가 느린 경우에도 나타날 수 있습니다.

# 4. 503 Service Unavailable

500 오류 메시지와 마찬가지로 또 다른 서버 측 오류는 503 Service Unavailable 입니다.

사용자가 액세스하려는 웹 사이트가 다운된 경우 서버 자체에서 이 메시지로 응답합니다. 그들은 이에 대해 아무 것도 할 수 없으므로 일반적으로 사이트 액세스 시도를 포기합니다. 이것이 사이트의 가동 시간을 모니터링하는 것이 매우 중요한 이유입니다.

# 5. 400 Bad Request

400 Bad Request 오류의 요지는 웹 서버가 사용자에게 액세스 요청이 잘못되었음을 알리는 것입니다.

URL 혹은 파라미터 등을 잘못 입력했을 수 있습니다. 다른 이유는 특정 페이지가 공개되지 않았기 때문일 수 있습니다. 이 메시지를 받은 사용자는 URL, Parameters, Query, Payload 등을 주의 깊게 확인해야 합니다.

# 6. 403 Forbidden

403 오류는 사용자가 사이트의 금지된 디렉토리에 액세스하려고 할 때 발생합니다. 페이지에 로그인 기회가 없으면 이 메시지가 나타납니다.

하지만 사이트 사용자의 문제를 해결할 수 있는 방법이 있습니다. 올바른 접근 권한이 있는지 확인하여 문제를 수정하십시오. Linux 파일의 경우 파일 및 폴더의 소유자 및 그룹 조정이 포함됩니다.

잘못 명명된 홈페이지는 오류의 또 다른 일반적인 원인입니다. 인덱스 페이지가 없을 수도 있습니다. 홈페이지가 index.html 또는 index.php인지 확인하여 이를 확인할 수 있습니다.

또 다른 가능한 원인은 비어 있는 http 문서 디렉토리입니다. 사이트의 콘텐츠를 올바른 디렉토리에 업로드하십시오. example.com을 사이트 이름으로 바꿔야 합니다.

# 7. 401 Unauthorized

401 Unauthorized라는 메시지는 403 오류와 유사합니다. 이는 사용자가 페이지를 보려면 사용자 이름과 비밀번호로 로그인해야 함을 의미합니다. Authorization required 라는 메시지로 나타날 수도 있습니다.

사용자 이름과 암호가 이미 입력된 경우 자격 증명이 유효하지 않을 수 있습니다.

시도해야 하는 첫 번째 수정 사항은 페이지가 올바른 URL에 있는지 확인하는 것입니다. 사이트 소유자는 해당 페이지를 가리키는 링크를 확인할 수 있습니다. 사용자가 클릭한 링크가 승인된 사용자만 액세스할 수 있는 잘못된 URL을 가리킬 수 있습니다.

사용자는 메인 페이지로 이동하여 로그인 링크를 찾을 수도 있습니다. 계정이 없으면 새 계정을 설정할 수 있습니다.

# 귀하의 사이트에서 웹사이트 오류 메시지를 제거하세요

이제 웹사이트 오류 메시지를 처리하는 데 필요한 정보를 얻으셨기를 바랍니다. 사용자에게 최상의 경험을 제공하려면 사이트를 하루 24시간 가동하는 것이 중요합니다.

혼자서 할 수 있는 일은 많지 않습니다. 도움이 필요하면 무료 원격 서비스를 제공할 수 있습니다. 웹 사이트를 모니터링하여 가동 중지 시간을 제거하므로 지금 확인하십시오.
